§ 06  /  FAQ

Questions we hear most.

What is CatalEx?

CatalEx is an AI operating layer for AI-native companies. It is a single platform where organizations build, deploy, measure, and govern every AI agent — whether embedded in a product, running autonomously in the background, or working alongside employees as named AI assistants. It reduces time to first production agent from 6–8 weeks to minutes.

I can build agents with coding tools like Claude Code. Why do I need CatalEx?

Coding agents help you build an agent. CatalEx helps you run, measure, and govern agents at organizational scale. Without a platform layer:

  • No memory or context — every session starts from scratch. CatalEx agents carry persistent memory and evolve with your business.
  • No self-evolution — agents perform the same on day 100 as day one. CatalEx agents learn from outcomes and get better over time.
  • No cost control — teams adopt AI tools and burn through budgets with no visibility into what the spend produced. CatalEx routes actions to the right model and cuts token spend by up to 90%.
  • No business KPIs — agents are deployed and forgotten. CatalEx ties every agent to measurable outcomes.
  • No guardrails — data leaks to cloud AI with no audit trail. CatalEx ships governance on day one.

CatalEx is model-agnostic — works across Claude, GPT, Gemini, and Llama with no vendor lock-in.

What are the three deployment modes?
  • Embed — agents run inside your product via API, handling tasks like onboarding verification and compliance checks in real time.
  • Autonomous — agents work in the background on scheduled or long-running tasks like invoice reconciliation, cohort reports, and lead outreach.
  • Assistants — named AI teammates (Analyst, SDR, Support Lead, Recruiter, Researcher, and more) that live in Slack, Notion, Jira, and GitHub and learn how each person works.
How does CatalEx reduce AI costs?

CatalEx cuts token spend by up to 90% through three mechanisms:

  • Code caching — caches the code agents write and re-runs it with new parameters instead of regenerating.
  • Smart model routing — routes each action to the most cost-effective model for the task.
  • Isolated execution — agents run in their own environments, preventing redundant processing.

This can result in up to 10× lower token cost on typical workloads.

Is CatalEx enterprise-ready?

Yes. CatalEx is built for teams whose agents touch regulated data, customer money, or production systems.

  • Role-based access — permissions inherited from your existing identity provider.
  • Full audit trail — every decision, tool call, and approval logged and queryable.
  • On-prem & in-region — deploy in your cloud, your VPC. Data never leaves your perimeter.
  • Human-in-the-loop — define which actions need approval, which run free, and which can never happen.
What are virtual employees?

Named AI assistants that join your team with a specific role. They carry persistent memory, evolve with how you work, and act proactively within guardrails you set.

  • Pre-built roles — Analyst, SDR, Support Lead, Recruiter, Researcher, and more.
  • Your tools — live in Slack, Notion, Jira, GitHub, and Google Drive.
  • They learn — adapt to how each person writes, decides, and escalates. They get better the longer they work with you.
  • Custom roles — define new ones in plain language.
Can I deploy CatalEx on-premises?

Yes. CatalEx supports on-premises and in-region deployment — in your cloud, your region, your VPC.

  • Your data never leaves your perimeter and never trains any model.
  • Suitable for regulated industries including financial services, healthcare, and government.
